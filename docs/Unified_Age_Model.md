# 统一年龄分组模型（老人/成人合并）

## 1. 业务背景
为了简化家庭健康管理应用的逻辑，降低用户认知负担，同时由于目前老人与成人在血脂、血糖等代谢类指标的参考规则、数据展示逻辑上完全一致，决定将原有的“老人 (senior)”类型彻底合并入“成人 (adult)”。

## 2. 年龄分组规则（最新）

| 类型 | 规则 (年龄 age) | 主要关注指标 | 备注 |
| :--- | :--- | :--- | :--- |
| **儿童 (child)** | `age < 18` | 身高、体重、视力、眼轴 | 启用发育百分位计算 |
| **成人 (adult)** | `age >= 18` | 血糖、血脂、体重 | 原老人类型合并至此 |

## 3. 逻辑变更点
1. **API 层级**: `member_type` 枚举仅保留 `child` 和 `adult`。
2. **判定逻辑**: 任何 18 岁及以上成员在系统内部统一标记为 `adult`。
3. **UI 表达**: 在所有页面文案、卡片标签、过滤器中移除“老人”称呼，统一使用“成人”。
4. **数据存储**: 数据库 `member_profiles` 表的 `member_type` 字段不再接受 `senior` 值。

## 4. 影响范围
- 成员档案创建/编辑流
- Dashboard 指标筛选逻辑
- 后端 Pydantic Schema 校验
- 自动化测试用例 (Happy Path/Boundary)
